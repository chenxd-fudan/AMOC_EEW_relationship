# Code and processed data for reproducing the analyses

This repository provides the custom code and processed intermediate data needed to reproduce the analyses and figures. The original model outputs are not included because they are too large for a code-availability package. The original CMIP6 and NAHosMIP outputs are publicly available from the official CMIP6/ESGF data portals.

## Repository contents

```text
.
├── data/                         Processed intermediate data used by the plotting scripts
├── code/                         MATLAB plotting scripts and preprocessing shell scripts
├── README.md                     Main repository guide
├── data/README_data.md           Description of processed data files
├── code/README_code.md           Description of analysis and plotting scripts
└── LICENSE                       License information
```

## Data availability

The processed data in the `data/` directory are sufficient to reproduce the figures and quantitative analyses generated by the scripts in this repository. The original model outputs are too large to be redistributed here. The preprocessing scripts document how the processed data were generated from the original model outputs.

## Software requirements

The scripts were developed using MATLAB and shell/CDO-based preprocessing workflows. The main plotting scripts require the following MATLAB helper functions or packages:

- `m_map`
- `near1`
- `addcolorplus`
- `rgb`
- `stipple`
- `cbarrow`
- `m_vec`

The preprocessing scripts require common climate-data command-line tools such as CDO and NCO.

## How to reproduce the figures

1. Clone this repository.
2. Make sure the processed `.mat` files are available in the `data/` directory.
3. Open MATLAB from the repository root.
4. Run the desired plotting script in `code/`.
5. If a script contains a line such as `dir = '';`, set it to:

```matlab
dir = './data/';
```

or run the script from the `data/` directory after adding the code directory to the MATLAB path.

## Notes on large files

The `.mat` files are processed intermediate data. If any `.mat` file is larger than GitHub's standard file-size limit, use Git LFS or deposit the data in a data repository such as Zenodo, Figshare, Dryad, OSF, or an institutional repository and provide the DOI here.

## Suggested citation

Please cite the associated paper and this repository when using the code or processed data.
